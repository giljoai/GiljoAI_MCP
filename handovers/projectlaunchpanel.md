# explaining the project Launc Panel

The top part of this panel which is nested into the normal website shows Project name being worke don, Project ID and Product name which the project belongs to.

# Orchestrator card
The orchestrator card should be a templated card , in the info field it should show this orchestrators ID (unique for each project) and a breif description of the role.  Here is where the copy prompt button lives to trigger the initial orchestrator

# USers Project description field

- Below the orchestrator card is the users original description with a scroll bar if it is long.
- Edit button if user wants to tune it last minute, and a [Save button] if they do edit it.
- The main mission window will get populated as the orchestrator defines the mission (which is its job after the prompt has been copied and pasted) here the develoepr should see how the orchestrator finalizes the mission and context used etc, the user should be able to read this and require a scroll bar if it is long.
- As the orchestrator works ans assignes agents, this is where the agent cards show up.  The agent name, the agent ID, the role it has and the assigned mission (via an eyebal view icon)
- once the develoepr agrees to all the roles, they click [accept mission]
