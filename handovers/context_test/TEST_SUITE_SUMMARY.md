## API Endpoints Used

1. **`PUT /api/users/me/field-priority`**
   - Updates field priority configuration
   - Accepts v2.0 format with priorities dict

2. **`PUT /api/users/me/context/depth`**
   - Updates depth configuration

3. **`POST /mcp`** (MCP HTTP endpoint)
   - Calls `get_orchestrator_instructions` tool

## Implementation Notes

### Design Decisions

1. **Sweep-based Testing** (not full combinatorial)
   - Full combinatorial: 4^8 × 4×4×5×2 = ~16 million combinations (infeasible)
   - Sweep-based: 57 meaningful tests covering all edge cases
   - Provides complete coverage in practical runtime

2. **Async/Await Architecture**
   - Uses `httpx.AsyncClient` for concurrent API calls
   - Proper connection pooling and cleanup
   - Rate limiting with configurable delays

3. **Configuration Format Conversion**
   - Internal: `{"toggle": bool, "priority": int}`
   - API: `{"version": "2.0", "priorities": {field: int}}`
   - Automatic conversion: `toggle: false` → `priority: 4`

4. **Error Handling**
   - Graceful failure handling
   - Detailed error messages
   - Continues testing on individual failures
   - Comprehensive error logging

### Test Strategy

- **Independence**: Each test is independent (can run in any order)
- **Idempotency**: Tests can be re-run without side effects
- **Isolation**: Each test sets complete configuration (no state leakage)
- **Validation**: Each test validates output matches input
