## Lessons From This Sprint (Read Carefully)

### Worker stall recovery

The 0750b worker stalled for ~50 minutes during pytest execution
(likely hung on DB connection tests). The orchestrator committed the
partial work (file deletions) and re-launched a second worker with the
remaining scope (CORE_TESTS.md creation and fixture cleanup). This is
the correct recovery pattern. Do not wait indefinitely.

### Chain log updates are unreliable

Worker agents sometimes do not update `chain_log.json`. Use `git log`
and `git status` as secondary progress indicators. If the worker
committed with the right message format, the phase is done regardless
of chain_log state.

### The dependency graph is a MAP, not a VERDICT

The graph contains 948 orphans. Most are entry points, scripts, CLI
tools, standalone utilities, or test files. Every file flagged by the
graph must be individually investigated before any action. "Orphan"
means zero import dependents -- it does NOT mean safe to delete.

### JS frontend dict-return patterns are intentional

The frontend composable pattern uses dict-like return objects. This is
standard Vue.js/JavaScript convention. Do not flag these as violations
of the Python dict-return rule.

### Handover precision matters

Vague handovers produce vague results. Include specific file paths,
line numbers, grep commands, and exact commit message formats. The
0750a and 0750b handovers are your template -- follow their level of
specificity.
