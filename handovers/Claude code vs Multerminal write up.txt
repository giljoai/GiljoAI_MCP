  Key Insight: User COPY/PASTE is the Bridge

  The server cannot push to user terminals. User must:
  1. Copy prompt from browser (Webpage UI)
  2. Paste into terminal (Claude Code CLI)

  ---
  Two Modes: Claude Code CLI vs Multi-Terminal

  | Aspect          | Claude Code CLI Mode                         | Multi-Terminal Mode             |
  |-----------------|----------------------------------------------|---------------------------------|
  | Terminals       | 1 (orchestrator uses Task tool)              | N (one per agent)               |
  | Agent spawning  | Orchestrator → Task(subagent_type=...)       | User copies each agent's prompt |
  | Play button (>) | Only orchestrator gets implementation prompt | Each agent card has play button |
  | Target users    | Claude Code users                            | Other AI tools (no Task tool)   |

  ---
  Corrected Two-Stage Flow (Claude Code CLI Mode)

  Stage 1: STAGING (🚀 Launch Tab)

  User: Copies "Stage Project" prompt → Pastes into Claude Code terminal
                      ↓
  Orchestrator (fresh session):
    1. health_check()
    2. get_orchestrator_instructions(orch_id, tenant_key)
       → Gets: project context, vision, agent templates list
    3. Analyzes requirements
    4. Creates mission plan
    5. update_project_mission(project_id, mission)
    6. spawn_agent_job() for each specialist
       → Creates jobs in "waiting" status
    7. send_message(broadcast: "STAGING_COMPLETE")
                      ↓
  Output: Mission saved in DB, N agent jobs created (status: "waiting")
  UI: "Launch Jobs" button appears → navigates to {} IMPLEMENT dashboard

  Stage 2: IMPLEMENTATION ({} IMPLEMENT Tab / Dashboard)

  User: Clicks play (>) on Orchestrator card → Copies implementation prompt
  User: Pastes into Claude Code terminal (FRESH SESSION - worst case)
                      ↓
  Orchestrator (assumes fresh, but might be same session):
    "Hey, you are orchestrator {orch_id, project_id, tenant_key}
     You have PREVIOUSLY:
     - Created a project mission
     - Spawned agents and assigned each a job

     If you need to re-read mission: {mcp_call_to_get_mission}
     If you need more context: get_orchestrator_instructions(...)

     YOUR JOB NOW:
     1. Create TODO list of agents to launch
     2. Update dashboard with task count: report_progress(...)
     3. For each agent, spawn via Task tool:
        Task(
          subagent_type='{agent_type}',  // MUST match .claude/agents/{agent_type}.md
          prompt='You are {agent_name}.
                  Call get_agent_mission(job_id=\"{job_id}\", tenant_key=\"{tenant_key}\")
                  to fetch your mission and begin execution.'
        )
     4. Monitor progress: receive_messages(), get_workflow_status()
     5. When all complete: complete_job()"
                      ↓
  Each Task subagent (in hidden subprocess):
    1. get_agent_mission(job_id, tenant_key)
       → Gets mission + full_protocol (6-phase lifecycle)
       → Transitions job from "waiting" → "working"
    2. Executes work per mission
    3. report_progress() for dashboard updates
    4. send_message() to orchestrator
    5. complete_job() OR report_error() (→ "completed"/"failed"/"blocked")

  ---
  Agent Job States (from your answer to Q3)

  waiting → working → completed
                   ↘ failed
                   ↘ blocked

  - waiting: Created by spawn_agent_job() in Stage 1
  - working: Transitioned by get_agent_mission() first call
  - completed/failed/blocked: Set by agent via MCP tools

  Belt-and-suspenders: State transition instructions are in:
  1. Implementation prompt (tells agent to call get_agent_mission)
  2. .claude/agents/{agent_type}.md template (agent behavior)

  ## IDENTITY

  - Created project mission
  - Spawned {N} specialist agents
  - Each agent has a job in "waiting" status

  ## YOUR OBJECTIVE
  Launch all specialist agents to implement the project.

  ## AGENT JOBS TO LAUNCH
  | Agent Name | Agent Type | Job ID |
  |------------|------------|--------|
  | {agent_name_1} | {agent_type_1} | {job_id_1} |
  | {agent_name_2} | {agent_type_2} | {job_id_2} |
  ...

  ## EXECUTION STEPS

  ### Step 1: Plan
  Create a TODO list. Update dashboard:
  report_progress(job_id='{orch_job_id}', progress={

    "total_steps": {N},
    "completed_steps": 0,
    "current_step": "Launching agents"
  })

  ### Step 2: Launch Each Agent via Task Tool
  For each agent above:
  Task(
    subagent_type="{agent_type}",  // MUST match template filename exactly
    prompt="You are {agent_name} for GiljoAI project.
            Call get_agent_mission(job_id='{job_id}', tenant_key='{tenant_key}')
            to fetch your mission and begin execution."

  - get_workflow_status(project_id='{project_id}', tenant_key='{tenant_key}')

  ### Step 4: Complete
  When all agents report completion:
  complete_job(job_id='{orch_job_id}', result={"summary": "...", "agents_completed": N})

  ## MCP TOOLS AVAILABLE
  - get_orchestrator_instructions() - Refresh full context
  - report_progress() - Update dashboard
  - receive_messages() - Check agent messages
  - get_workflow_status() - Monitor all agents
