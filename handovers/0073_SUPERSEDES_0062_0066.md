## The Definitive Vision

### Core Concept: Static Agent Grid
```
Multi-terminal orchestration with manual control, not automated pipeline
```

### Key Principles
1. **Agents are orchestrated manually** via terminal prompts, not automated
2. **Status reflects current activity**, not pipeline stage
3. **Messages are MCP communications** between developer and agents
4. **Orchestrator manages context and coordination**, not just messaging
5. **Multi-tool support is essential** (Claude Code, Codex, Gemini)

### The 7 Agent Status States

| Status | Meaning | Who Uses |
|--------|---------|----------|
| **Waiting** | Ready to launch | All agents start here |
| **Preparing** | Loading context | Agent acknowledged, loading mission |
| **Working** | Executing tasks | Active work (testing for test agents) |
| **Review** | Under review | Awaiting approval/feedback |
| **Complete** | Mission done | Successfully finished |
| **Failed** | Error occurred | Unrecoverable error |
| **Blocked** | Waiting for input | Needs external dependency |

### Orchestrator Special Status
- Always shows: **"Context Management & Project Coordination"**
- Never changes (orchestrator is always active)
- Has dual copy prompts (Claude Code + Codex/Gemini)

### MCP Message Center
- **Right panel** (30% width, permanent)
- **Chronological feed** of ALL agents
- **[MCP] prefix** on all messages (clarifies these are MCP messages)
- **Broadcast capability** to all agents
- **Developer messages** (right-aligned, blue)
- **Agent messages** (left-aligned, grey)

### Copy Prompt Architecture

#### Orchestrator (Special)
- **Button 1**: "Copy Prompt (Claude Code)" - For Claude's subagent automation
- **Button 2**: "Copy Prompt (Codex/Gemini)" - For manual terminal orchestration

#### Regular Agents
- **Single Button**: "Copy Prompt" - Universal format works in any terminal
- Tooltip: "For Codex/Gemini or Claude if used with individual terminal windows"

### Tab Structure
1. **Launch Panel** - Setup and mission configuration
2. **Orchestration** - Active agent grid and messaging (NOT "Active Jobs" or "Jobs")

## MCP Tools Required

### Status Management
```bash
mcp set_agent_status --status=waiting|preparing|working|review|complete|failed|blocked
mcp set_agent_status --progress=45  # For working status
mcp set_agent_status --reason="error description"  # For failed/blocked
```

### Message Management
```bash
mcp send_mcp_message --target=orchestrator --content="message"
mcp send_mcp_message --target=broadcast --content="message to all"
mcp send_mcp_message --target=agent --agent_id=UUID --content="message"
mcp read_mcp_messages --unread_only
```

## Key Differences from Previous Vision

### What 0062/0066 Got Wrong
1. **Kanban implies automation** - Reality is manual orchestration
2. **Column movement** - Agents don't "flow", they change status
3. **Per-agent messages** - Need unified view of all communications
4. **Missing multi-tool support** - No CODEX/GEMINI prompts
5. **No closeout workflow** - Projects couldn't be properly completed

### What 0073 Gets Right
1. **Static grid** - Matches mental model of terminals
2. **Status badges** - Clear visual indication without movement
3. **Unified messaging** - See all agent communications
4. **Multi-tool prompts** - Proper support for all terminals
5. **Complete lifecycle** - From launch to closeout
