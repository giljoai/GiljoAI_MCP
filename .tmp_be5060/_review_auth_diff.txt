    feat(API-0021b): scope-aware MCP tool gating
    
    Edition Scope: CE
    
    Adds scope-based authorization to the MCP transport. Every @mcp.tool is
    classified into mcp:read (8), mcp:write (9), or mcp:agent (18) via the
    TOOL_SCOPES sidecar registry. JWT callers see only tools whose registered
    scope is in the token's scope set; mcp:agent tools (spawn_job, complete_job,
    send_message, mission edits, ...) are unreachable via OAuth /authorize at
    issuance time AND at the tools/list / tools/call boundaries. API-key auth
    bypasses entirely (full surface).
    
    Changes:
    
    - src/giljo_mcp/auth/jwt_manager.py: optional scope kwarg on
      create_access_token, written into the JWT payload when set.
    - src/giljo_mcp/services/oauth_service.py: OAUTH_GRANTABLE_SCOPES
      ({mcp:read, mcp:write}) and DEFAULT_OAUTH_SCOPE constants;
      validate_authorize_request rejects scope tokens outside that set;
      exchange_code_for_token forwards auth_code.scope into the JWT.
    - api/endpoints/oauth.py: AuthorizeRequest.scope default -> mcp:read mcp:write;
      /.well-known/oauth-protected-resource scopes_supported sourced from
      OAUTH_GRANTABLE_SCOPES.
    - api/endpoints/mcp_sdk_server.py: TOOL_SCOPES dict + helpers; re-registers
      the lowlevel server's ListToolsRequest and CallToolRequest handlers with
      scope-aware filter and dispatch gate; MCPAuthMiddleware stamps
      scope[state][auth_method] and [scopes] at JWT and api_key success branches.
      JWTs without a scope claim default to [mcp:read, mcp:write] so legacy and
      cookie tokens still work.
    
    Tests (boundary regression at the FastMCP transport layer per CLAUDE.md):
    
    - tests/integration/test_mcp_scope_filtering.py (new): 14 tests covering
      middleware stamping, list_tools filter, call_tool dispatch gate, OAuth
      scope rejection, exchange-forwards-scope, registry completeness.
    - tests/api/test_oauth_audience_binding.py: adds R8 protected-resource
      scopes_supported assertion.
    - tests/test_oauth.py + tests/api/test_oauth_endpoints.py: updates existing
      fixtures from the deprecated scope=mcp default to mcp:read mcp:write.
    
    Server-side enforcement on tools/call is mandatory (defense-in-depth): a
    client crafting a hidden-tool dispatch via JWT receives a ToolError instead
    of silent execution.
