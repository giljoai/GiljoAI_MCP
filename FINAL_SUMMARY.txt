HOW IT WORKS
=============================================================================

User Workflow (No Page Refresh Required):

1. User opens Settings -> Context Priority Configuration tab
   - Git History is disabled (shows alert)

2. User navigates to Settings -> Integrations tab

3. User toggles "Enable Git Integration" ON
   - Backend processes toggle

4. Backend emits WebSocket event: 'product:git:settings:changed'
   - Event contains: { product_id, settings: { enabled: true } }

5. ContextPriorityConfig component receives WebSocket event (< 50ms)
   - handleGitIntegrationUpdate() is called
   - gitIntegrationEnabled reactive ref updates to TRUE
   - Template reactively updates:
     * Alert disappears (v-if="!gitIntegrationEnabled" re-evaluates)
     * Git History controls enable (:disabled="isContextDisabled('git_history')" becomes false)

6. RESULT: User sees changes IMMEDIATELY, NO PAGE REFRESH NEEDED

Data Flow:
  Integrations Tab
        |
        v
  Backend API (/git-integration)
        |
        v
  ProductService.update_git_integration()
        |
        v
  WebSocket: 'product:git:settings:changed'
        |
        v
  Context Tab WebSocket Listener
        |
        v
  handleGitIntegrationUpdate()
        |
        v
  gitIntegrationEnabled.value = true
        |
        v
  Vue Reactivity triggers template update

CODE CHANGES SUMMARY

Added to ContextPriorityConfig.vue:

1. Import WebSocket composable:
   import { useWebSocketV2 } from '@/composables/useWebSocket'

2. Initialize WebSocket handler:
   const { on, off } = useWebSocketV2()

3. New handler function:
   function handleGitIntegrationUpdate(data) {
     if (!data || !data.settings) return
     const newState = data.settings.enabled || false
     gitIntegrationEnabled.value = newState
     // ... logging and UI feedback
   }

4. Register listener in onMounted:
   on('product:git:settings:changed', handleGitIntegrationUpdate)

5. Cleanup in onUnmounted:
   off('product:git:settings:changed', handleGitIntegrationUpdate)
