# GiljoAI MCP Distribution Manifest
# This file lists all files and directories to include in a distribution package

## Core Application Files (REQUIRED)
/src/                       # Core Python application
/api/                       # REST API and WebSocket implementation
/frontend/                  # Vue.js web interface
/tests/                     # Test suite

## Configuration Files (REQUIRED)
/config.yaml.example        # Configuration template
/.env.example               # Environment variables template
/requirements.txt           # Python dependencies
/package.json               # Node.js dependencies (if exists in frontend)
/pyproject.toml            # Python project configuration
/setup.py                  # Python package setup
/alembic.ini              # Database migration configuration

## Installation Files (REQUIRED)
/INSTALL.md                # Installation guide
/README.md                 # Project documentation
/quickstart.bat           # Windows quick setup
/quickstart.sh            # Unix/Mac quick setup

## Optional Directories (RECOMMENDED)
/scripts/                  # Setup and utility scripts
/examples/                 # Usage examples
/migrations/              # Database migrations (if using Alembic)
/docker/                  # Docker configuration (if exists)

## Development Files (RECOMMENDED FOR DEVELOPERS)
/.ruff.toml               # Python linting configuration
/.eslintrc.json          # JavaScript linting (if exists)
/.prettierrc             # Code formatting (if exists)
/mypy.ini               # Type checking configuration (if exists)

## Files to EXCLUDE from Distribution
/.git/                    # Git repository data
/.github/                 # GitHub specific files
/Docs/Sessions/          # Development session logs
/Docs/devlog/            # Development logs
/backups/                # Database backups
/__pycache__/            # Python cache (all locations)
/*.pyc                   # Compiled Python files
/node_modules/           # Node dependencies (rebuild locally)
/.pytest_cache/          # Test cache
/.coverage               # Coverage reports
/data/                   # Local data files
/logs/                   # Log files
/*.log                   # Log files
/.env                    # Local environment file (not .env.example)
/config.yaml            # Local configuration (not config.yaml.example)
/*.db                    # Database files
/*.db-shm               # SQLite temp files
/*.db-wal               # SQLite write-ahead log
/test_*.py              # Development test scripts in root
/*_test_*.json          # Test result files
/monitor*.py            # Development monitoring scripts
/fix_*.py               # Development fix scripts
/debug_*.py             # Debug scripts
/coverage*.json         # Coverage report files
/*_report.md            # Generated reports
/.mcp.json              # Local MCP configuration
/.mcp-serena.json       # Local Serena configuration
/.serena/               # Serena working directory
/.claude/               # Claude local settings

## Distribution Package Structure
When creating a ZIP/TAR distribution, the structure should be:

giljo-mcp/
├── src/
├── api/
├── frontend/
├── tests/
├── scripts/
├── examples/
├── config.yaml.example
├── .env.example
├── requirements.txt
├── setup.py
├── pyproject.toml
├── INSTALL.md
├── README.md
├── quickstart.bat
├── quickstart.sh
└── MANIFEST.txt

## Notes for Packagers
1. Ensure all Python files maintain their directory structure
2. Frontend assets should be included but node_modules excluded
3. Documentation should be in root directory
4. Configuration examples must have .example extension
5. No local data, logs, or database files should be included
6. Development-specific monitoring and test scripts should be excluded
