# Agent Setup Instructions for Other Dev Stations

**Date:** 2026-04-08
**Context:** The GiljoAI_MCP repo architecture has changed. Every dev station running `GiljoAI_MCP_Private` needs to be updated.

## What Changed

- `GiljoAI_MCP_Private` is now the **single source of truth** for all editions (CE, SaaS, Demo).
- `GiljoAI_MCP` (public repo) is now a **read-only downstream** -- generated by an export script, never edited directly.
- Three editions run from this one codebase via the `GILJO_MODE` env var (not set = CE, `demo` = Demo, `saas` = SaaS).

## Steps to Run on Each Dev Station

### 1. Pull latest from origin

```bash
cd /path/to/GiljoAI_MCP_Private
git pull origin master
```

This brings in:
- Updated `CLAUDE.md` with the new repo architecture rules
- `scripts/export_ce.sh` (CE export script)
- `.export-exclude` (export exclusion list)
- License headers on all Python files
- `src/giljo_mcp/licensing/` module (LicenseValidator)
- `docs/CODE_STANDARDS.md`
- Updated `CONTRIBUTING.md`, `README.md`, `api/app.py`

### 2. Add the public repo as a git remote (if the station has a local clone of GiljoAI_MCP)

```bash
git remote add public /path/to/GiljoAI_MCP
```

This remote is only used for cherry-picking community PRs. If this station does not have a local clone of the public repo, skip this step -- the export script is only run from the primary dev station.

### 3. Verify the setup

```bash
# License validator should work
python -c "from src.giljo_mcp.licensing.validator import LicenseValidator; r = LicenseValidator().validate(); print(r.message)"
# Expected: GiljoAI MCP Community Edition — single-user use only.
```

### 4. Read the updated CLAUDE.md

The `CLAUDE.md` in the repo root has been rewritten. It now contains:
- Two-repo architecture (private = source of truth, public = CE downstream)
- Edition isolation rules (CE, SaaS, Demo directory separation)
- The Deletion Test (delete saas/ and demo/ dirs, CE must still work)
- Export script usage instructions
- Community PR cherry-pick workflow

**You must follow these rules going forward.** The key points:

- **All development happens in this repo.** Never commit to the public repo.
- **CE code** goes in standard directories (`api/`, `src/giljo_mcp/`, `frontend/src/`).
- **SaaS code** goes in `saas/`, `saas_endpoints/`, `saas_middleware/`, `frontend/src/saas/`.
- **Demo config** goes in `demo/`.
- CE code **never** imports from `saas/` or `demo/`.
- The `GILJO_MODE` env var controls which edition runs at startup.

## What NOT to Do

- Do NOT clone or work in `GiljoAI_MCP` (public) for development.
- Do NOT create separate repos for SaaS or Demo editions.
- Do NOT run the export script (`scripts/export_ce.sh`) unless you are on the primary dev station and intentionally pushing a CE release.

## Questions?

If anything is unclear, read `CLAUDE.md` in the repo root -- it is the authoritative reference. The `docs/EDITION_ISOLATION_GUIDE.md` has detailed rules for where code goes.
