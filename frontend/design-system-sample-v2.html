17. Accessibility
WCAG AA compliance rules. All text must meet 4.5:1 contrast on its surface. Touch targets minimum 44px.
Prohibited Patterns
class="text-medium-emphasis" Vuetify class — renders #666 which fails AA
color: #5a6a80 Old muted — 2.74:1 FAIL
color: #999 Legacy token — unreliable contrast
border: 1px solid ... on rounded cards Use smooth-border class instead
variant="outlined" on v-card Use variant="flat" + smooth-border
v-tabs / v-tab Use pill-button toggle pattern
border-radius: 50% on agent badges Use border-radius: 8px (square tinted)
Required Patterns
class="text-muted-a11y" #8895a8 — 4.98:1 on #182739
color: var(--text-secondary) #a3aac4 — 6.56:1
getAgentColor() from agentColors.js All 6 pass AA on #182739
hexToRgba() from colorUtils.js For tinted backgrounds
getAgentBadgeStyle() from colorUtils.js For agent badge inline styles

23. Toast / Notification Patterns
Toast notifications via useToast() composable. Four types: success, error, warning, info. All use showToast({ message, type, timeout }).

Toast Types

Project created successfully
Failed to save settings. Check your connection and try again.
Connection lost. Attempting to reconnect…
Agent configuration updated

showToast API

Import
import { useToast } from '@/composables/useToast'
const { showToast } = useToast()
Usage Examples
// Success (auto-dismiss, default timeout)
showToast({
  message: 'Project created successfully',
  type: 'success'
})

// Error with recovery suggestion
showToast({
  message: 'Failed to save. Try again.',
  type: 'error',
  timeout: 8000
})

// Warning / Info
showToast({ message, type: 'warning' })
showToast({ message, type: 'info' })
Message Guidelines
TypePattern
success"<Entity> <action>ed successfully"
error"Failed to <action>. <Recovery suggestion>."
warningPlain language, state what happened
infoNeutral confirmation of state change
24. Form Validation Patterns
Vuetify form validation using :rules bindings. Field-specific error messages appear below the input. Use v-form with ref for programmatic validation.

Input States

Letters, numbers, and hyphens only
Letters, numbers, and hyphens only
Name contains invalid characters. Use letters, numbers, and hyphens only.
Name is available

Vuetify Rules Pattern

Rule Definitions
const rules = {
  required: (v) => !!v || 'Field is required',
  email: (v) => /.+@.+\..+/.test(v) || 'Invalid email',
  minLength: (v) => v.length >= 8 || 'Min 8 characters',
}
Template Usage
<v-form ref="formRef">
  <v-text-field
    v-model="name"
    label="Project Name"
    :rules="[rules.required]"
  />
  <v-text-field
    v-model="email"
    label="Email"
    :rules="[rules.required, rules.email]"
  />
</v-form>
Programmatic Validation
const { valid } = await formRef.value.validate()
if (valid) {
  // submit form
}
