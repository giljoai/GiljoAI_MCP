================================================================================
ORCHESTRATOR SPECIAL HANDLING
================================================================================

WHY NO TEMPLATE FOR ORCHESTRATOR?
├─→ Orchestrator is not an instantiable agent
├─→ Orchestrator is a system-level component
├─→ Orchestrator prompt is global configuration
└─→ Not stored in agent template database

TECHNICAL IMPLEMENTATION:
├─→ AgentDetailsModal detects: agent_type === 'orchestrator'
├─→ Calls different endpoint: /api/v1/system/orchestrator-prompt
├─→ No template_id field required
├─→ Prompt stored in separate system config (not templates table)
└─→ Can be overridden by admin in settings

API ENDPOINTS:
├─ GET /api/v1/system/orchestrator-prompt
│  └─→ Returns current prompt (default or override)
├─ PUT /api/v1/system/orchestrator-prompt
│  └─→ Save custom orchestrator prompt override
└─ POST /api/v1/system/orchestrator-prompt/reset
   └─→ Remove override, use default
