## Overview

Projects View v2 delivers a modern, accessible interface to manage projects with real-time search, status filters, sorting, and interactive status management. It respects single-active constraints and soft-delete recovery.

## Key Features

- Real-time search across name, mission, and ID
- Status filter tabs with dynamic counts: Active, Inactive, Completed, Cancelled, Deleted
- Multi-column sorting (name, status, dates)
- StatusBadge with interactive actions:
  - Activate (makes project active; enforces single-active per product)
  - Deactivate (replaces older “Pause” terminology per 0071)
  - Complete, Cancel
  - Restore (from Deleted/Completed to Inactive)
  - Delete (soft delete → 10-day recovery window)
- Deleted projects modal with restore
- Product isolation (lists only projects for current active product)
- Multi-tenant safe by design
- WCAG 2.1 AA accessibility, keyboard navigation

## Constraints & Interactions

- Single Active Project per Product (0050b): Only one project can be active per product.
- State Simplification (0071): Use Deactivate instead of Pause.
- Soft Delete (0070): Deleted projects are recoverable for 10 days, then purged.

## Usage Notes

- Use the search box to filter by name, mission keywords, or ID.
- Use status tabs to focus on specific states; counts update in real time.
- Click a project’s status badge to perform allowed transitions.
- Deleted projects appear in a dedicated view/modal; use Restore to recover.
